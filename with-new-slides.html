What we inherited

The 2-pizza team
was built for a different era.

For over a decade, the cross-functional scrum team was the gold standard of software delivery. 8-12 people spanning development, design, QA, DevOps, project management, and product ownership. The model worked — but it was designed around human limitations: coordination overhead, handoff latency, and specialization silos.

The math was simple: more complexity required more people. More people required more process. More process required more management. The team grew to absorb the friction.

The 2-pizza team
"We need a human for every role on the team."
8-12 people. Developers, designers, QA, DevOps, PM, PO, Scrum Master. Each role a specialist. Value delivered through coordination and handoffs.
The agentic team
"We need AI-enabled humans who can direct agents."
2-5 people. Each one a full-stack contributor augmented by AI agents that handle execution, testing, deployment, and review. Value delivered through leverage, not headcount.
What changed
Then foundation models changed the equation. AI can now write code, run tests, review PRs, and deploy — at machine speed, with human direction.
The AI-enabled engineer

One person,
an army of agents.

An AI-enabled engineer doesn't just use AI as a tool — they orchestrate a fleet of agents across every discipline. They write code through coding agents, validate through QA agents, deploy through DevOps agents, and research through analysis agents. One person now covers what used to require an entire team.

The human brings what agents can't: judgment, context, creativity, and customer understanding. They set direction, make trade-off decisions, and navigate ambiguity. The agents handle execution — at machine speed, at machine scale.

The measurement problem

Measuring for impact, not just activity.

Traditional teams tracked what was easy to count. But the metrics that actually indicate performance were rarely tracked — and when they were, the tooling was inconsistent at best.

SE
Software Engineer
Activity tracked
Story points PR throughput Lines of code
Measures busyness, not value. Estimates inflate, velocity varies wildly across teams, and none of it tells you if the feature actually shipped.
Impact metric
Cycle time
The only metric that answers "how long from idea to production?" Fast cycle time means fewer handoffs, less WIP, and faster feedback loops.
SA
Software Architect
Activity tracked
Design docs Tech debt scores
Counting documents produced says nothing about whether the decisions in them were correct. Most architecture quality is assessed by gut feel in retros.
Impact metric
Downstream rework rate
A good architectural decision holds up. A bad one causes rework weeks later. This metric traces implementation churn back to the decision that caused it.
QA
QA Tester
Activity tracked
Test cases written Coverage %
High coverage and lots of test cases can still miss critical bugs. Volume of tests doesn't equal quality of testing — shallow tests pass easily and catch nothing.
Impact metric
Defect detection rate
What percentage of bugs are caught before production? This is the only QA metric that directly measures whether testing is actually protecting users.
DO
DevOps Engineer
Activity tracked
Deploy frequency Provisioning time
Deploying often is good, but it says nothing about reliability. Teams can deploy ten times a day and still have systems that break constantly.
Impact metric
Recovery time
When something breaks in production, how fast is it fixed? This measures the thing customers actually feel — downtime — not how busy the pipeline looks.
PM
PM / PO / Scrum Master
Activity tracked
Sprint completion Overhead hours
Completing sprints on paper is easy when you cut scope or shift dates. These metrics reward process compliance, not actual delivery outcomes.
Impact metric
On-time delivery rate
Did the thing ship when you said it would? Predictability is the PM's real deliverable — stakeholders plan around your commitments.
UX
UI/UX Designer
Activity tracked
Screens per sprint Design system coverage
Producing screens quickly means nothing if they require endless revision cycles. Output speed masks design quality problems until they surface in development.
Impact metric
Iteration cycles to approval
How many rounds of revision before stakeholders sign off? Fewer cycles means the designer understood the problem upfront and nailed the solution early.

Three impact metrics
The Agentic Engineer's goal is fast, clean delivery. Three metrics tell you if they're achieving it.
Feature Cycle Time — how fast does work move from first commit to production? Fewer handoffs mean faster flow. QA Pass Rate — is speed coming at the cost of quality? Builds must pass automated checks before they ship. Deploy Stability — did the work hold up? No rollback within 48 hours means it landed clean.

In the old model, correlating a developer's code to its QA outcome and deploy stability required tracing across three separate roles. The Agentic Engineer owns all three — making the correlation automatic.

THE FORMULA
Feature Cycle Time + QA Pass Rate + Deploy Stability
= Fast, Clean Delivery
Feature Cycle Time

Source

Tools

Target

Source
Deployment logs + incident/rollback events
Tools
PagerDuty, Opsgenie, ArgoCD, or deploy pipeline flags
Target
≥90% of deploys survive 48h without rollback

Agentic metrics

The pattern repeats across every role.

Each agentic role follows the same principle: ignore the agent-amplified volume, measure the end-to-end outcome that only the role owner can see.

Agentic Architect
Arch + Tech Lead + Security
Noise (agent-amplified)
ADRs produced, code reviews completed, security scans run, perf profiles
Signal
Decision durability
Correlate ADRs in Confluence to downstream rework, security remediations, and perf fixes in Jira. AI can auto-tag the relationships.
Agentic PO/Designer
PO + UX + BA
Noise (agent-amplified)
Research reports generated, design concepts, specs written, competitive analyses
Signal
Spec-to-build fidelity
Count clarification requests, design revisions, and scope changes post-handoff via Jira comment threads and Figma version history.
Agentic QA
Dedicated quality (larger teams)
Noise (agent-amplified)
AI-generated test cases, visual regressions executed, perf benchmarks run
Signal
Escaped defect cost
Production incidents (PagerDuty) weighted by severity and business impact. Sum of (severity x impact weight) per release, trending down.
Agentic Delivery Engineer
Sprint coord + stakeholders + release
Noise (agent-amplified)
Status reports auto-generated, stakeholder updates sent, blockers logged
Signal
Delivery predictability index
Standard deviation of sprint completion rates over a rolling 6-sprint window. Low variance = calibrated commitments + effective blocker removal.
